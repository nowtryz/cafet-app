Le premier couple de balises `<!--CONFIG > <CONFIG-- >` de chaque mail sera interprété en JSON et supprimé avant l'envoi. Ce commentaire a pour but de modifier les configuration propres à ce mail. Les variables suivantes sont toutes optionnelles mais peuvent être utilisée.

L'utilisation de variable est possible en utilisant des accolades : {nomDeLaVariable}, elle est ensuite modifiable via $mail->setVar("nomDeLaVariable", "valeur")

Le mail doit etre encodé en UTF-8.
